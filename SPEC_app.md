# App Router 構造仕様書 (app フォルダ)

このプロジェクトの Next.js App Router 構成と各ファイルの役割、コンポーネントの性質（Server/Client）を以下にまとめます。

## フォルダ構造概要

```text
app/
├── favicon.ico
├── globals.css
├── layout.tsx
└── page.tsx
```

## 各ファイルの詳細

| パス | 役割 | コンポーネント種別 | 備考 |
| :--- | :--- | :--- | :--- |
| `layout.tsx` | ルートレイアウト | **Server Component** | アプリケーション全体の基本構造（HTML, Bodyタグ）、フォント設定、メタデータ定義、グローバルCSSのインポートを担当。 |
| `page.tsx` | メインページ (/) | **Client Component** | ダメージ計算シミュレーターのメイン画面。入力状態の管理、シミュレーションの実行、表示レイアウトの切り替えなどのロジックが集約されている。 |
| `globals.css` | グローバルスタイル | - | Tailwind CSS のインポート、ダークモード用 CSS 変数、スクロールバーのカスタマイズなどを定義。 |
| `favicon.ico` | アイコン | - | ブラウザのタブ等に表示されるアイコン。 |

## ルーティング

| パス | 対応ファイル | 説明 |
| :--- | :--- | :--- |
| `/` | `app/page.tsx` | アプリケーションの唯一のエントリポイント。全機能（設定・計算結果表示）がこのページ内で完結する。 |

## 設計上の特徴

### Client Component の集中
`app/page.tsx` に `"use client"` が指定されており、React の `useState` や `useEffect` を用いた動的な状態管理と計算ロジックがクライアント側で動作するように設計されています。これは、ユーザーの入力（ステータスやバフの変更）に対して即座にシミュレーション結果を反映させるための選択です。

### レイアウトの役割分担
- **layout.tsx**: SEO用のメタデータ（タイトル、説明）や、アプリケーション全体のフォントスタイルなど、静的な枠組みを定義します。
- **page.tsx**: 具体的な計算機能、タブ切り替え、分割表示モードなどの動的な UI インタラクションを担当します。

### ダークモード対応
`globals.css` と `page.tsx` の状態管理によって、Tailwind CSS の `dark:` クラスを用いたテーマ切り替えが実装されています。
